# HC Request System - Design Context

## Design Context

### Users
- **Managers**: ยื่นคำขออัตรากำลัง (Headcount) ใหม่หรือทดแทน ต้องการกรอกข้อมูลได้รวดเร็วและติดตามสถานะได้ง่าย
- **TA/HR**: รับเคส อัพเดทสถานะการรับสมัคร ดูภาพรวมทั้งหมด ต้องการข้อมูลครบถ้วนและจัดการได้สะดวก

### Brand Personality
**Professional, Efficient, Trustworthy**

- Professional: ดูเป็นระบบ enterprise ที่ใช้งานจริงในบริษัท
- Efficient: เน้นความเร็วในการทำงาน ไม่ต้องคลิกเยอะ
- Trustworthy: ข้อมูลชัดเจน น่าเชื่อถือ ไม่ทำให้สับสน

### Aesthetic Direction
**Modern Corporate with Fresh Energy**

- ใช้สีเขียวของ Freshket (#008065, #00ce7c) เป็นหลัก แต่ไม่จัดจ้านเกินไป
- เน้น whitespace และ spacing ที่สบายตา
- Typography ชัดเจน อ่านง่าย
- มี micro-interactions เล็กน้อยเพื่อให้รู้สึก "มีชีวิต" แต่ไม่รบกวนการใช้งาน
- รองรับทั้ง Light และ Dark mode

### Design Principles

1. **Clarity First**: ข้อมูลต้องเห็นชัดเจนในหน้าเดียว ไม่ต้องคลิกเยอะ
2. **Speed Matters**: ทุก interaction ต้องรวดเร็ว ไม่มี delay ที่รู้สึกได้
3. **Progressive Disclosure**: ซ่อนข้อมูลรองไว้ แสดงเฉพาะสิ่งสำคัญก่อน
4. **Consistent Patterns**: ใช้รูปแบบเดียวกันทั้งระบบ ไม่ทำให้ผู้ใช้งง
5. **Accessible by Default**: ตัวอักษรอ่านง่าย สี contrast เพียงพอ รองรับผู้ใช้ทุกคน

### References
- Linear.app: สำหรับความเร็วและความ clean
- Notion: สำหรับการจัดข้อมูลที่เป็นระเบียบ
- Vercel Dashboard: สำหรับ modern corporate feel

### Anti-References
- ไม่เอาแบบเก่าๆ แบบ Windows 95 / legacy enterprise
- ไม่เอาแบบสีสันจัดจ้านเกินไป ดูไม่ professional
- ไม่เอาแบบที่ต้องคลิกเยอะๆ เพื่อทำงานง่ายๆ
